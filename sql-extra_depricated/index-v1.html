Datenbankschema
Drei Tabellen bilden die Grundlage aller Abfragen in diesem Kurs.
Hinweis: Die Tabelle subject enthält eine direkte teacher_id (1:1-Beziehung). Zusätzlich existiert die N:m-Brückentabelle subject_teacher, über die einem Fach mehrere Lehrer zugeordnet werden können. Diese doppelte Zuordnung ist bewusst gestaltet und ist Gegenstand mehrerer Aufgaben.
▤ teacher
PK teacher_id INT
first_part VARCHAR(80)
last_part VARCHAR(80)
▤ subject
PK subject_id INT
subject_name VARCHAR(100)
FK teacher_id INT
▤ subject_teacher (N:m)
PK FK subject_id INT
PK FK teacher_id INT
