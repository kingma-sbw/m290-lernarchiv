Alle Begriffe auf einen Blick
1 · Benutzer erstellen
CREATE USER

Benutzer werden über Benutzername und Host eindeutig identifiziert. Der Host bestimmt, von wo der Zugriff erlaubt ist.

CREATE USER 'max'@'localhost' IDENTIFIED BY 'sicheres_passwort';
CREATE USER 'max'@'%' IDENTIFIED BY 'passwort';  -- % = jeder Host
2 · Verfügbare Rechte
PrivilegBedeutung
ALL PRIVILEGESAlle Rechte auf einer Ebene (ausser GRANT OPTION)
SELECTDaten lesen
INSERTDaten einfügen
UPDATEDaten ändern
DELETEDaten löschen
CREATEDatenbanken / Tabellen anlegen
DROPObjekte löschen
GRANT OPTIONRechte an andere Benutzer weitergeben
3 · Rechte vergeben und entziehen
GRANT – Rechte erteilen

Die Ebene bestimmt den Gültigkeitsbereich: Datenbank, Tabelle oder Spalte.

-- Datenbank-Ebene: alle Tabellen einer DB
GRANT SELECT, INSERT ON meinedatenbank.* TO 'max'@'localhost';

-- Tabellen-Ebene: nur eine bestimmte Tabelle
GRANT UPDATE ON meinedatenbank.kunden TO 'max'@'localhost';

-- Spalten-Ebene: nur bestimmte Spalten
GRANT SELECT (name, email) ON meinedatenbank.kunden TO 'max'@'localhost';

-- Änderungen aktivieren
FLUSH PRIVILEGES;
REVOKE – Rechte entziehen
REVOKE INSERT ON meinedatenbank.* FROM 'max'@'localhost';
4 · Praxisbeispiel
Zwei Benutzer für eine Datenbank

Benutzer mit Lese- und Schreibrechten:

CREATE USER 'redakteur'@'localhost' IDENTIFIED BY 'lesen_schreiben123';
GRANT SELECT, INSERT, UPDATE, DELETE ON meinedatenbank.* TO 'redakteur'@'localhost';

Benutzer mit nur Schreibrechten (kein SELECT):

CREATE USER 'schreiber'@'localhost' IDENTIFIED BY 'nur_schreiben456';
GRANT INSERT, UPDATE ON meinedatenbank.* TO 'schreiber'@'localhost';

FLUSH PRIVILEGES;

Der Benutzer schreiber kann Daten einfügen und ändern, aber nicht lesen — SELECT wurde bewusst nicht erteilt.
